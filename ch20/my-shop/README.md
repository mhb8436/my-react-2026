## 미니 프로젝트 아이디어

쇼핑몰 변형 상세
1. PC 견적 조립기 — CPU/메인보드/램/GPU 카테고리별로 부품 목록을 보여주고 각 카테고리에서 하나씩만 고르게 합니다. 장바구니가 "배열 push"가 아니라 "카테고리 키로 덮어쓰기"가 되면서 상태 구조를 다시 생각하게 됩니다. 여력이 되면 CPU 소켓과 메인보드 소켓 불일치 경고까지.

2. 배달 주문앱 — 같은 메뉴라도 옵션이 다르면 다른 항목이어야 합니다. id만으로 중복 판정하면 깨지는 걸 학생이 직접 겪게 되는 게 이 주제의 핵심입니다.

3. 좌석 예매 — 5×8 좌석 그리드, 일부는 이미 예매됨(disabled), 최대 4석. 2차원 배열을 map 두 번으로 렌더링하고 불변성 지키며 갱신하는 연습이 확실히 됩니다.

4. 도서관 대출 — 가장 안전한 주제입니다. 진도가 느린 학생에게 배정하기 좋습니다. 대출/반납 버튼이 도서 상태에 따라 바뀌는 조건부 렌더링, 3권 한도 검증이 전부입니다.

5. 여행 경비 플래너 — 1일차/2일차/3일차 탭을 두고 각 일자에 항목을 담습니다. 상태가 { day1: [], day2: [] } 형태로 한 겹 깊어지는 게 포인트라, 실력 있는 학생에게 주면 좋습니다.

퀴즈 변형 상세
6. 성향 테스트 — 채점 로직이 정답 비교가 아니라 축별 점수 누적(E/I, S/N …)이고, 마지막에 유형 코드로 결과를 찾습니다. 결과를 URL 쿼리로 공유하는 것까지 하면 라우팅 연습이 붙습니다.

7. 제한시간 타이머 퀴즈 — 이 주제 하나가 useEffect cleanup을 제대로 가르칩니다. 문제가 바뀔 때 타이머를 안 걷어내면 바로 버그가 눈에 보이기 때문에 학습 피드백이 즉각적입니다.

8. 플래시카드 암기 — 틀린 카드를 큐 뒤에 다시 넣어서 전부 맞힐 때까지 도는 구조. localStorage로 진도 저장까지 붙이면 새로고침 대응 개념도 잡힙니다.

9. 국기/도시 맞추기 — restcountries 같은 공개 API에서 데이터를 받아 보기 4개를 랜덤 생성합니다. Promise/async, 로딩 스피너, 에러 화면 세 가지 분기를 반드시 다뤄야 해서 fetch 학습에는 이게 제일 낫습니다. 다만 API 응답 구조 파악에 시간이 걸리니 2일 중 반나절을 여기에 쓸 각오가 필요합니다.

10. 자격증 모의고사 — 과목별로 채점하고 "평균 60점 이상, 과목별 40점 미만 없음" 같은 합격 판정을 넣습니다. 결과 화면에서 오답만 필터링해 해설을 보여주는 게 메인.
